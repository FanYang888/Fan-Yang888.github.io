react生命周期

react生命周期

一、初始化阶段

  • componentWillMount
  • render
  • componentDidMount

componentWillMount:第一次上树前的最后一次修改状态的机会。 （不安全）

应用：初始化数据。

render：渲染页面，dom树，不允许render中改变state状态值，会陷入死循环。

componentDidMount：页面dom节点渲染完毕。

应用：1.数据请求 axios；

​ 2.订阅函数调用，订阅方法见“react组件通信方法”一文；

​ 3.定时器 setInterval；

​ 4.基础创建完的dom进行初始化，。。如BetterScroll。

二、运行中阶段

  • componentWillReceivePeops : 父组件修改属性触发，可以最先获得父组件传来的属性，可以利用属性进行ajax或者逻辑处理。
  • shouldComponentUpdata(nextProps,nextState): 该函数里面this.state是旧的函数值，返回false会阻止更新；（***性能优化函数***）
  • componetWillUpdata：（不安全）只能拿到旧的状态值，不能更新状态，因为只要setstate，就又要执行componetWillUpdata => render => componentDidUpdata这个过程，又将陷入死循环。
  • render： 只能访问this.props和this.state，不允许修改状态和dom输出,会死循环；
  • **componentDidUpdata(prevProps,prevState)**： 该函数里面this.state是新的状态值，可以修改dom，可以传参数获取旧状态值。

应用：

1：componentDidUpdata会多次执行，为了避免多次执行，可以给该方法传递参数，通过判断参数中旧状态的值来决定是否执行某段代码。例如：

2.shouldComponentUpdata(nextProps,nextState)用于判断是否更新，当状态值没变的时候控制不更新，可以优化速度。

三、销毁阶段

  • componentWillUnmount: 在删除组件之前进行清理操作，如计时器和事件监听器。

四、新生命周期

老生命周期的问题：

(1) componentWillMount ,在ssr中 这个方法将会被多次调用， 所以会重复触发多遍，同时在这里如果绑定事件，将无法解绑，导致内存泄漏 ， 变得不够安全高效逐步废弃。

(2) componentWillReceiveProps 外部组件多次频繁更新传入多次不同的 props，会导致不必要的异步请求

(3) componetWillupdate, 更新前记录 DOM 状态, 可能会做一些处理，与componentDidUpdate相隔时间如果过长， 会导致状态不可信。

新生命周期的替代

（1）getDerivedStateFromProps（nextProps,nextState） 第一次的初始化组件以及后续的更新过程中(包括自身状态更新以及父传子)都会触发 ，返回一个对象作为新的state，返回null则说明不需要在这里更新state；

getDerivedStateFromProps可以代替componentWillMount 和componentWillReceiveProps 。

(2) getSnapshotBeforeUpdate 取代了 componetWillUpdate ,触发时间为update发生的时候，在render之后dom渲染之前返回一个值，该函数return返回的值作为componentDidUpdate的第三个参数。

附：性能优化函数案例shouldComponentUpdate

效果：输入框输入值，对应下标值的框改变颜色

注意：此时效果可以实现，但是每次更改输入框的值，十个框都会重新渲染一遍，浪费内存。如下图所示：

期待效果：每次输入框的值改变以后，只移除原来那个框的红色样式和给新框添加上红色边框样式，而其余的不重新渲染。

实现方法：在子组件中通过shouldComponentUpdate控制当前组件是否渲染。

效果：只会在第一次渲染到页面的时候会执行十次render函数，后面更改输入框的值的时候，只会执行旧框和新选择框的render函数。

性能优化 方案二 PureComponent

PureComponent是react官方提供的，会自动比较新旧props、state（值相等,或者对象含有相同的属性、且属性值相等 ），决定shouldcomponentUpdate 返回true 或者false， 从而决定要不要呼叫 render function。

使用方法：引用的时候import React {PureComponet} from react即可。

注意：如果你的 state 或 props 『永远都会变』，那 PureComponent 不适合，因为 shallowEqual 也需要花时间。
