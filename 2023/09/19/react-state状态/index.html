6.状态重置问题

详解：

第一点：例如一个计数组件NumberCount，用isshow和&&来控制该组件的显示与隐藏{isshow && <NumberCount>}，每次隐藏以后再显示count的值都是从默认值开始。原因就是因为当isshow位false的时候，组件被销毁，对应的状态被重置。

第二点：同样上面的技术组加NumberCount，用isstyle判断是否显示样式

1，{isstyle?<NumberCount style = {{border: "1px solid red"}}:<NumberCount>},此时组件位置没有发生改变，状态会保留。

2，{isstyle?<NumberCount style = {{border: "1px solid red"}}:<div><NumberCount></div>}，此时状态会被重置，因为处于不同的结构体中。

3，{isstyle?<NumberCount key=‘key1’ style = {{border: "1px solid red"}}:<div><NumberCount key=‘key2’></div>}，此时状态会被重置，因为key属性不同。
